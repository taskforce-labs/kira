# Domain Modelling

## Overview
Create domain models and context maps to articulate the domain and context of the product.

## Steps

1. **Discover Domain Language**
   - Identify entities, aggregates, and value objects
   - Document ubiquitous language
   - Map domain events

2. **Context Mapping**
   - Identify bounded contexts
   - Map relationships (partnership, shared kernel, customer-supplier, conformist, ACL, OHS)
   - Document context map

3. **Create Artifacts**
   - Domain model diagrams or descriptions
   - Context map
   - Store in configured artifact location

## Output

Domain model and context map artifacts. Use the domain-discovery skill for detailed guidance.
