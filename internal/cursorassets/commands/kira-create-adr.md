## Overview
Generate Architecture Decision Records (ADRs) to document significant technical or architectural decisions.

## Steps

1. **Context**
   - Describe the context and problem
   - Identify stakeholders and constraints

2. **Decision**
   - State the decision clearly
   - Outline options considered and rationale

3. **Consequences**
   - Document positive and negative consequences
   - Note follow-up actions or reviews

4. **Create Artifact**
   - Use project ADR format and location (e.g. docs/adr/)
   - Store with date and ID

## Output

An ADR document. Use the technical-discovery skill for ADR structure and when to create ADRs.
